# Android SDK Setup Guide

## Problem
The build is failing with: **"SDK location not found"**

This happens when Gradle can't find your Android SDK installation.

## Quick Fix (Automatic)

### Option 1: Run Setup Script (Recommended)

**Windows PowerShell:**
```powershell
.\setup-android-sdk.ps1
```

**Windows Command Prompt:**
```cmd
setup-android-sdk.bat
```

The script will:
1. Automatically detect your Android SDK location
2. Create the `android/local.properties` file
3. Optionally set ANDROID_HOME environment variable

### Option 2: Manual Setup

1. **Find your Android SDK location:**
   - Open Android Studio
   - Go to: **File > Settings > Appearance & Behavior > System Settings > Android SDK**
   - Copy the "Android SDK Location" path
   - Common locations:
     - `C:\Users\YourUsername\AppData\Local\Android\Sdk`
     - `C:\Android\Sdk`

2. **Create `android/local.properties` file:**
   ```
   sdk.dir=C\:\\Users\\YourUsername\\AppData\\Local\\Android\\Sdk
   ```
   **Important:** Use double backslashes (`\\`) in the path!

3. **Set ANDROID_HOME (Optional but recommended):**
   
   **Windows (PowerShell - Current Session):**
   ```powershell
   $env:ANDROID_HOME = "C:\Users\YourUsername\AppData\Local\Android\Sdk"
   ```
   
   **Windows (Permanent - User):**
   ```powershell
   [System.Environment]::SetEnvironmentVariable("ANDROID_HOME", "C:\Users\YourUsername\AppData\Local\Android\Sdk", "User")
   ```
   
   **Or via System Settings:**
   1. Press `Win + X` and select "System"
   2. Click "Advanced system settings"
   3. Click "Environment Variables"
   4. Under "User variables", click "New"
   5. Variable name: `ANDROID_HOME`
   6. Variable value: Your SDK path (e.g., `C:\Users\YourUsername\AppData\Local\Android\Sdk`)
   7. Click OK

## Verify Setup

1. **Check local.properties exists:**
   ```bash
   type android\local.properties
   ```
   Should show: `sdk.dir=C\:\\Users\\...`

2. **Check ANDROID_HOME (if set):**
   ```powershell
   echo $env:ANDROID_HOME
   ```

3. **Try building again:**
   ```bash
   npm run android
   ```

## Troubleshooting

### Issue: "Path does not exist"
- Make sure Android Studio is installed
- Verify the SDK path is correct
- Check that the path uses forward slashes or escaped backslashes

### Issue: "Still getting SDK not found"
1. Close and reopen your terminal
2. Clean the build:
   ```bash
   cd android
   ./gradlew clean
   cd ..
   ```
3. Try building again

### Issue: "Permission denied"
- Run PowerShell/Command Prompt as Administrator
- Or manually create the `local.properties` file

## Example local.properties

```
## This file is automatically generated.
## Do not modify this file -- YOUR CHANGES WILL BE ERASED!
##
sdk.dir=C\:\\Users\\JohnDoe\\AppData\\Local\\Android\\Sdk
```

**Note:** The path format uses double backslashes (`\\`) because Gradle needs escaped backslashes.

## After Setup

Once `local.properties` is created, you should be able to build:

```bash
npm run android
```
