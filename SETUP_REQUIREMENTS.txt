                        SETUP INSTRUCTIONS
================================================================================

STEP 1: VERIFY ALL SOFTWARE IS INSTALLED
----------------------------------
✓ Node.js v24.5.0 - INSTALLED
✓ npm v11.5.1 - INSTALLED
✓ Git v2.51.2 - INSTALLED
✓ HeidiSQL - INSTALLED
✓ MySQL - VERIFY (check if running)

TO CHECK MYSQL:
- Open HeidiSQL
- Try connecting to localhost with credentials
- If connection fails, install MySQL or start MySQL service


STEP 2: CREATE DATABASE
----------------------------------
1. Open HeidiSQL
2. Connect to MySQL:
   - Hostname: localhost
   - User: root
   - Password: root (or your MySQL password)
   - Port: 3306

3. Create new database:
   - Right-click on left panel
   - Select "Create new" > "Database"
   - Name: SkillSphere_Db
   - Collation: utf8mb4_general_ci
   - Click OK


STEP 3: SETUP BACKEND
----------------------------------
1. Open Terminal/Command Prompt
2. Navigate to backend folder:
   cd D:\skillsphere\AppAndroidSS\skillsphere_app\skillsphere_app\backend

3. Install backend dependencies:
   npm install

4. Check .env file (backend\.env):
   MYSQL_DB=SkillSphere_Db
   MYSQL_USER=root
   MYSQL_PASSWORD=root
   MYSQL_HOST=localhost
   MYSQL_PORT=3306
   PORT=5000
   JWT_SECRET=your_super_secret_jwt_key_change_this_in_production

5. Create initial Super Admin account:
   npm run seed

6. Start backend server:
   npm run dev

   Expected output:
   - "Server running on port 5000"
   - "Database connected successfully"


STEP 4: SETUP FRONTEND (WEB APP)
----------------------------------
1. Open NEW Terminal/Command Prompt
2. Navigate to frontend folder:
   cd D:\skillsphere\AppAndroidSS\skillsphere_app\skillsphere_app\AppAndroidSS

3. Install frontend dependencies:
   npm install

4. Start web development server:
   npm run web

   Or for development mode:
   npm run web:dev

5. Open browser (Chrome 143.x recommended) and go to:
   http://localhost:8080


STEP 5: VERIFY SETUP
----------------------------------
1. Backend health check: http://localhost:5000/health
   Should return: {"status":"OK","message":"Server is running"}

2. Frontend: http://localhost:8080

================================================================================
                        COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE 1: "npm is not recognized"
SOLUTION:
- Reinstall Node.js v24.5.0
- Restart computer
- Verify: node --version


ISSUE 2: "MySQL connection error"
SOLUTION:
- Check MySQL service is running
- Verify credentials in backend\.env
- Test connection in HeidiSQL first


ISSUE 3: "Port 5000 already in use"
SOLUTION:
- Kill process:
  netstat -ano | findstr :5000
  taskkill /PID [process_id] /F
- Or change PORT in backend\.env


ISSUE 4: "Port 8080 already in use"
SOLUTION:
- Webpack auto-selects next available port
- Check terminal for actual port number


ISSUE 5: "Module not found"
SOLUTION:
- Delete node_modules and package-lock.json
- Run: npm install
- Restart terminal


ISSUE 6: Webpack build errors
SOLUTION:
- Clear cache: npm cache clean --force
- Delete node_modules
- Run: npm install
